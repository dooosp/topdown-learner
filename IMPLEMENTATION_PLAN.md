## 개선 항목 (우선순위순)

### 1. PIN 모달 컨텍스트 제공
**문제:** 비밀번호 팝업이 왜 뜨는지 설명 없음

- 모달 상단에 앱 타이틀 + 한줄 설명 추가
- "이 앱은 소크라테스식 학습 도구입니다"
- placeholder: "관리자에게 받은 PIN 입력"

### 2. 온보딩 웰컴 섹션
**문제:** 첫 진입 시 빈 화면, 뭘 해야 할지 모름

PIN 인증 후 → 모드 선택 전 웰컴 가이드 표시:

```
┌─────────────────────────────────┐
│  TopDown Learner                │
│  제1원리부터 시작하는 학습       │
│                                  │
│  ① 주제를 입력하면              │
│  ② AI가 핵심 원리를 추출하고    │
│  ③ 소크라테스식 질문으로 안내   │
│                                  │
│  [주제 학습]  [코드 분석]        │
└─────────────────────────────────┘
```

- 3단계 플로우 시각적 가이드
- localStorage `topdown_onboarded` 플래그로 재방문 시 스킵

### 3. 버튼 레이블 명확화
**문제:** "일반 주제", "내 코드" 등 추상적

| 현재 | 개선 |
|------|------|
| 일반 주제 | 주제 학습 |
| 내 코드 | 코드 분석 |
| 탐구 시작 | 학습 시작 |
| 코드 학습 | 분석 시작 |

### 4. placeholder 가독성 개선
**문제:** 입력 필드 placeholder가 흐릿함

- `::placeholder` 색상 불투명도 0.3 → 0.5
- topicInput: "예: 반도체, 양자역학, 머신러닝..."
- chatInput: "질문이나 답변을 입력하세요"

### 5. 예시 주제 칩
**문제:** 주제 입력란에 아무 힌트 없음

입력란 아래 클릭 가능한 예시 칩 4개:

```
[반도체] [양자역학] [TCP/IP] [미적분]
```

- 클릭 시 topicInput에 자동 입력
- 학습 시작 후 숨김 처리

## 수정하지 않는 것

| 항목 | 이유 |
|------|------|
| 다크 테마 자체 | 리뷰에서도 "색감 괜찮다" 평가 |
| 2분할 레이아웃 | 학습 도구로서 적합한 구조 |
| 백엔드 API | 프론트엔드 UX 문제이므로 불필요 |
| 퀴즈/PDF 기능 | 학습 시작 후 기능은 정상 동작 |

## 자가 비판 (Phase 3)

**취약점:**
1. 온보딩 반복 노출 → localStorage 플래그로 "다시 보지 않기" 처리
2. 예시 칩 분야 편향 → 과학/공학/수학/인문에서 각 1개 선정
3. 모바일 온보딩 레이아웃 → 반응형 처리 필수

**리스크:**
- 기존 localStorage 진도 데이터 충돌 → 온보딩은 별도 키(`topdown_onboarded`) 사용
- style.css 변경 → 신규 클래스만 추가, 기존 클래스 수정 최소화
